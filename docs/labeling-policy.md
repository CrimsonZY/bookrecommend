# 标注规则（Labeling Policy）v1

适用对象：`data/books_enriched.json` 这类书籍条目中的 `mood_tags` / `scene_tags` / `style_tags`。

目标：从“示例修订集（B）相对未修订集（A）”的变化中抽象出**可泛化**的自动标注规范，用于批量标注剩余数据。

---

## 1. mood_tags 归一化规则

- **受控词表优先**
  - `mood_tags` 必须来自固定情绪词表（示例：温暖/治愈/希望/平静/浪漫/幽默/哲思/热血/悲伤/沉重/孤独…）。
  - 不在词表内的近义词、口语词、组合词，必须先映射为词表内“标准词”。

- **去泛化：限制「治愈」滥用**
  - `治愈` 仅在整体基调明确“安抚、松弛、积极回甘”时保留。
  - 若作品主导基调为苦难/悲剧/压抑/创伤叙事，则优先标注 `悲伤` / `沉重`（并**移除** `治愈`）。

- **主情绪优先（主次分层）**
  - 默认输出 3 个：主情绪 1–2 个 + 次级情绪/余韵 1 个。
  - 不建议把互相稀释的情绪同时堆叠为“都对”。

- **情绪与叙事类型协同（补充型规则）**
  - 明显成长/冒险/对抗叙事：可补 `热血`。
  - 明显观念讨论/思辨：可补 `哲思`。
  - 明显轻松表达/喜剧气质：可补 `幽默`。

---

## 2. style_tags 归一化规则

### 2.1 标签分槽（强制）

为避免风格标签堆叠与“串位”，`style_tags` 按 3 个槽位组织：

- **体裁位（Genre slot）**：小说/散文/人物传记/科幻…（原则上只保留 1 个）
- **主题位（Theme slot）**：社会观察/哲学思考/爱情/历史…（可 1–2 个）
- **文学位（Literary slot，可选）**：文学（用于表达“文学性/严肃叙事”的统一信号）

### 2.2 归一化映射（来自 A→B 的稳定规律）

- **评价/地位类降维**
  - `经典文学` 统一归一为 `文学`（“经典”是评价，不是风格）。

- **成长类标签弱化/拆解**
  - 不直接使用 `成长小说` 作为风格标签。
  - 用“体裁位 + 主题位 + 文学位（可选）”表达，例如：`小说` +（可选）`文学` +（可选）`哲学思考`/`社会观察` 等。

- **散文标签更克制**
  - `散文` 仅在文本形态明确为随笔/短章体/非虚构散记时使用。
  - 若只是“文笔好/文学性强”，用 `文学` 表达，不用 `散文` 兜底。

- **上位词优先（可泛化）**
  - 当同一维度出现多个近义/同族标签时，优先保留更稳定、覆盖更广的上位词（例如 `文学`），避免过细碎的长尾词。

---

## 3. scene_tags 归一化规则

- **只描述“阅读发生条件”**
  - `scene_tags` 只描述读者阅读的时间/空间/节奏条件：通勤/碎片阅读/周末下午/长期阅读/睡前…。
  - 不允许把“用途/动机/购买意图”塞进 `scene_tags`。

- **「送礼」处理（强约束）**
  - `送礼` 默认不标注；只有当作品强信号体现“礼物属性”（例如礼物书/赠礼定位/节日赠送强相关内容与文案）才允许标注。
  - 否则一律移除（A→B 变化显示该标签被系统性清退）。

---

## 4. 标签冲突处理规则

- **mood_tags 冲突（治愈 vs 悲伤/沉重）**
  - 若作品整体压抑/悲剧为主：保留 `悲伤`/`沉重`（二选一或主次），移除 `治愈`。
  - 若作品整体温暖回甘：保留 `治愈`，`悲伤/沉重` 至多保留其一且作为次级。

- **style_tags 冲突（体裁位唯一）**
  - `小说/散文/人物传记/科幻` 等体裁位标签原则上只保留 1 个主标签。
  - 其余信息用主题位表达（社会观察/哲学思考/爱情/历史…）。

- **维度不串位（硬规则）**
  - scene_tags 不承载用途/动机；style_tags 不承载评价/地位；mood_tags 不承载题材/主题。
  - 发现串位词：优先删除，而不是“兼容进来”。

---

## 5. 标签数量约束规则

- **mood_tags**
  - 默认 3 个（主 1–2 + 次 1），上限 4。

- **scene_tags**
  - 默认 3 个（高频场景 + 节奏场景 + 时间段/沉浸度场景），上限 4。
  - 严禁用途型词混入（如“送礼”除非强信号）。

- **style_tags**
  - 默认 3–4 个（体裁位 1 + 主题位 1–2 + 文学位可选 1），上限 4。
  - 避免 5+ 堆叠；优先做归一化与分槽后再裁剪。
